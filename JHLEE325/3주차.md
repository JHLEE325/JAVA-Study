## 1. 산술 연산자

자바에서 산술 연산자는 수학적 계산을 수행하는 기본 연산자이며, <br>
JVM 바이트코드 레벨에서도 가장 기본적인 연산 단위다. <br>


### 산술 연산자의 종류
| 연산자  | 의미   | 예시           |
| ---- | ---- | ------------ |
| `+`  | 덧셈   | `a + b`      |
| `-`  | 뺄셈   | `a - b`      |
| `*`  | 곱셈   | `a * b`      |
| `/`  | 나눗셈  | `a / b`      |
| `%`  | 나머지  | `a % b`      |
| `++` | 1 증가 | `a++`, `++a` |
| `--` | 1 감소 | `a--`, `--a` |

<br>

### 알아둘만한 포인트
#### 모든 정수 연산은 최소 `int` 단위로 수행
```java
byte a = 10;
byte b = 20;
// byte c = a + b; - 컴파일 에러
int c = a + b;
```
JVM의 산술 연산 명령어가 `int` 단위이기 때문

#### 피연산자 중 더 큰 타입으로 승격
```
byte → short → int → long → float → double
```

#### 나눗셈(`/`)에 관하여
- 정수 나눗셈 : 소수점 이하 **버림**
- 실수 나눗셈 : 피연산자 중 하나라도 실수면 실수로 연산
  ```java
  double b = 5 / 2.0; // 2.5
  ```

#### 0으로 나누는 경우
- 정수형 : ArithmeticException (런타임 예외 발생)
- 실수형 : IEEE 754 규칙을 따름
  ```java
  double x = 10.0 / 0; 
  // Infinity
  
  double y = 0.0 / 0;
  // NaN
  ```

#### 문자열과 `+` 연산자
```java
String s = "Hello" + 10;  // "Hello10"
// 아래처럼 변환
new StringBuilder()
    .append("Hello")
    .append(10)
    .toString();
```

#### 오버플로우
자바는 정수 오버플로우를 체크하지 않음 -> 예외가 발생하지 않고 오버플로우되서 넘어감

## 2. 비트 연산자

비트 연산자는 정수형 데이터를 비트 단위(0,1)로 직접 조작하는 연산자 <br>
2의 보수 / 시프트 / 부호 확장 / >>> 차이 등이 중요

<br>

### 비트 연산자의 종류
| 연산자   | 의미            | 설명          |                |
| ----- | ------------- | ----------- | -------------- |
| `&`   | AND           | 둘 다 1이면 1   |                |
| `     | `             | OR          | 둘 중 하나라도 1이면 1 |
| `^`   | XOR           | 서로 다르면 1    |                |
| `~`   | NOT           | 비트 반전       |                |
| `<<`  | 왼쪽 시프트        | 비트를 왼쪽으로 이동 |                |
| `>>`  | 오른쪽 시프트       | 부호 유지 시프트   |                |
| `>>>` | 부호 없는 오른쪽 시프트 | 0으로 채움      |                |

<br>

### 알아둘만한 포인트
#### 시프트 연산자
왼쪽 시프트(`<<`)
```java
3 << 2

00000011
<< 2
00001100 = 12
```
오른쪽 시프트 (`>>`) - 부호 유지
```java
-8 >> 2

-8 = 11111000
>> 2
11111110 = -2
```
부호 없는 오른쪽 시프트 (`>>>`)
```java
-8 >>> 2

11111000
>>> 2
00111110
```

#### 시프트 연산의 주의점
```java
int x = 1 << 32;  // 1
```
이유 : int는 32비트로 shift 값은 32로 나눈 나머지만 사용 <br>
long 인 경우 64로 남은 나머지 사용

#### 음수 표현 방법
절댓값의 이진수 -> 비트 반전(~) -> +1

## 3. 관계 연산자

관계 연산자는 두 피연산자의 대소 관계 또는 동일성 여부를 비교하여 `boolean`값을 반환하는 연산자

<br>

### 관계 연산자의 종류
| 연산자  | 의미     |
| ---- | ------ |
| `==` | 같다     |
| `!=` | 다르다    |
| `>`  | 크다     |
| `<`  | 작다     |
| `>=` | 크거나 같다 |
| `<=` | 작거나 같다 |

<br>

### 알아둘만한 포인트
#### 실수 비교
```java
double a = 0.1 + 0.2;
double b = 0.3;

a == b;  // false
```
IEEE 754 부동소수점 오차로 false 나옴

#### NaN 비교
```java
double x = Double.NaN;

x == x;  // false
```
NaN은 모든 비교 연산 결과가 false <br>
Double.isNaN(x) -> 이것만 true

#### 레퍼런스 타입에서의 비교
레퍼런스 타입의 `==`은 주소 비교
```java
String a = new String("hi");
String b = new String("hi");

a == b;        // false
a.equals(b);   // true
// StringPool은 예외
```

#### 객체 비교 시 Null인 경우
```java
String s = null;

s.equals("hi"); // NPE

"hi".equals(s);
Objects.equals(a, b);
```

#### 비교 연산과 자동 형변환
자동으로 큰 쪽으로 형변환이 된 후 값을 비교함
```java
int a = 10;
long b = 10L;

a == b;  // true
```

## 4. 논리 연산자
논리 연산자는 boolean 값을 대상으로 논리 연산을 수행하는 연산자 <br>
Short-Circuit, 평가 순서, &과 &&의 차이 등이 주요함

<br>

### 논리 연산자의 종류
| 연산자  | 의미              | 특징             |
| ---- | --------------- | -------------- |
| `&&` | 논리 AND          | Short-Circuit  |
| `\|\|` | 논리 OR | Short-Circuit |
| `!`  | 논리 NOT          | 값 반전           |
| `&`  | 논리 AND / 비트 AND | 항상 양쪽 평가       |
| `\|` | 논리 OR / 비트 OR   | 항상 양쪽 평가       |
| `^`  | 논리 XOR          | boolean XOR 가능 |

<br>

### Short-Circuit
#### AND(`&&`)
```java
false && (10 / 0 > 1)
// 오른쪽 실행 안함
```
AND는 왼쪽이 false면 전체가 false 확정

#### OR(`\|\|`)
```java
true || (10 / 0 > 1)
// 오른쪽 실행 안함
```
OR는 왼쪽이 true 면 전체 true 확정

<br>

### 알아둘만한 포인트
#### &과 &&의 차이
&&는 Short-Circuit 이기 때문에 왼쪽 항에 따라 오른쪽 실행 여부가 결정됨 <br>
&는 그런거 없이 양쪽 다 실행됨

#### boolean 에서의 XOR(`^`)
```java
true ^ false  // true
true ^ true   // false
```
다르면 true, 같으면 false

#### 평가 순서
자바는 왼쪽 -> 오른쪽 순서로 평가
```java
boolean result = method1() && method2();
```
method1 실행 -> false면 종료 <br>
method1 실행 -> true 면 method2 실행

## 5. `instanceof` 연산자

`instaceof`는 객체가 특정 타입의 인스턴스인지 런타임에 검사하는 연산자

<br>

### 기본 문법
```java
if (obj instanceof String) {
    System.out.println("문자열입니다.");
}
```
- 결과는 `boolean`
- 런타임 타입을 기준으로 판단

<br>

### 알아둘만한 포인트
#### null과 instanceof
```java
String s = null;

s instanceof String;  // false
```
null은 어떤 타입의 인스턴스도 아니다 <br>
예외는 발생하지 않고 false 반환

#### 인터페이스와 instanceof
```java
interface Animal {}
class Dog implements Animal {}

Animal a = new Dog();

a instanceof Dog;     // true
a instanceof Animal;  // true
```
인스턴스도 검사 가능

#### 제네릭과 instanceof
```java
List<String> list = new ArrayList<>();

list instanceof List<String>;  // 컴파일 에러
```
제네릭은 타입 소거 때문에 런타임에 타입 정보가 없음

#### 상속과 instanceof
```java
class A {}
class B extends A {}
class C extends B {}

A obj = new C();

obj instanceof A;  // true
obj instanceof B;  // true
obj instanceof C;  // true
```
하위 타입이면 모두 true

## 6. 대입 연산자

대입 연산자는 오른쪽 값을 계산한 뒤 왼쪽 변수에 저장하는 연산자

<br>

### 기본 대입 연산자 `=`
```java
int a = 10;
```
오른쪽 표현식 계산 -> 왼쪽 변수에 저장 (전체 표현식의 결과는 "대입된 값")
```java
int a;
int b = (a = 5);
System.out.println(b);  // 5
// a = 5 자체가 5를 반환
```

<br>

### 평가 순서
자바는 왼쪽 -> 오른쪽으로 평가
```java
arr[i] = i++;
```
실행과정
1. `arr[i]`의 주소 계산
2. `i++` 평가 (값은 기존 i, 이후 증가)
3. 대입 수행

<br>

### 복합 대입 연산자
| 연산자                  | 의미       |
| -------------------- | -------- |
| `+=`                 | 더해서 대입   |
| `-=`                 | 빼서 대입    |
| `*=`                 | 곱해서 대입   |
| `/=`                 | 나눠서 대입   |
| `%=`                 | 나머지 후 대입 |
| `&=`, `\|=`,`^=` | 비트 연산 후 대입 |
| `<<=`, `>>=`, `>>>=` | 시프트 후 대입 |

복합 대입 연산자를 수행할 때는 암묵적 캐스팅을 한다
```java
byte b = 10;
b += 20;   // 가능

byte b = 10;
b = b + 20;  // 컴파일 에러
```

<br>

### 알아둘만한 포인트
#### 체이닝 대입
```java
int a, b, c;
a = b = c = 10;
```
c=10 -> b=c -> a=b 순서로 대입됨

#### final 변수와 대입
final은 선언 이후 다시 대입하는 것이 불가 <br>
참조 타입의 경우 참조 변경이 불가
```java
final int x = 10;
x = 20;  // 컴파일 에러

final List<String> list = new ArrayList<>();
list.add("hi");  // 가능
list = new ArrayList<>(); // 불가능
```

#### 배열의 대입
```java
int[] arr1 = {1,2,3};
int[] arr2 = arr1;

arr2[0] = 99;
System.out.println(arr1[0]);  // 99
```
shallow copy 되어 같은 배열 취급 -> 값이 같이 바뀜

#### 증감 연산과 대입의 혼합
```java
int a = 1;
a = a++;
System.out.println(a);  // 1
```
a++ = 1을 반환하고 a=2로 증가하는데 반환한 1이 a에 다시 대입되므로 1이 출력됨

## 7. 화살표(`->`) 연산자 (Lambda Expression)

자바8부터 도입되었으며 람다 표현식을 정의하는 문법 <br>
기본적으로
```java
(parameters) -> expression
(parameters) -> { statements }
```
처럼 표현한다 <br>
스트림 API의 핵심이다

<br>

### 알아둘만한 포인트
#### 함수형 인터페이스
람다는 반드시 하나의 추상 메서드만 가진 인터페이스와 연결됨

#### 람다의 타입 추론
```java
Comparator<String> c = (a, b) -> a.compareTo(b);
```
람다는 타겟 타입이 있어야 함 -> 위에서의 타입은 `Comparator<String>`

#### 내부 동작 원리
- `invokedynamic` 바이트코드 사용
- 런타임에 동적으로 구현체 생성
- 별도의 .class 파일 생성하지 않음

#### 캡쳐링
람다는 외부 지역변수를 사용할 수 있으나 변경 불가능한 지역변수만 사용 가능하다
```java
int x = 10;

Runnable r = () -> {
    System.out.println(x);
};
```

## 8. 3항 연산자

3항 연산자는 조건식의 결과에 따라 두 표현식 중 하나를 선택하는 연산자 <br>
자바에서 유일한 3개의 피연산자를 가지는 연산자

<br>

### 기본 문법
```java
int a = 10;
int b = 20;

int max = (a > b) ? a : b;
```
조건식은 반드시 `boolean`임

<br>

### 알아둘만한 포인트
#### if-else 와의 차이
if-else는 statement 지만 3항 연산자는 표현식으로 값을 반환한다

#### 타입 결정 규칙
3항 연산자는 두 결과 표현식의 타입을 맞춰줘야 한다 <br>
타입이 다른 경우에는 자동 승격된다
```java
double result = true ? 1 : 2.0;
// double로 승격
```
```java
Integer a = null;
int b = 10;

int result = true ? a : b;
// 언박싱 후 NullPointerException 발생
```

## 8. 연산자 우선순위

| 순위 | 연산자                                                             | 설명                     |
| -- | --------------------------------------------------------------- | ---------------------- |
| 1  | `expr++` `expr--`                                               | 후위 증감                  |
| 2  | `++expr` `--expr` `+` `-` `~` `!`                               | 단항 연산                  |
| 3  | `*` `/` `%`                                                     | 곱셈, 나눗셈                |
| 4  | `+` `-`                                                         | 덧셈, 뺄셈                 |
| 5  | `<<` `>>` `>>>`                                                 | 시프트                    |
| 6  | `<` `<=` `>` `>=` `instanceof`                                  | 관계 연산                  |
| 7  | `==` `!=`                                                       | 동등 비교                  |
| 8  | `&`                                                             | 비트 AND                 |
| 9  | `^`                                                             | 비트 XOR                 |
| 10 | `\|`                                                            | 비트 OR                  |
| 11 | `&&`                                                            | 논리 AND (Short-Circuit) |
| 12 | `\|\|`                                                          | 논리 OR (Short-Circuit)  |
| 13 | `? :`                                                           | 3항 연산자                 |
| 14 | `=` `+=` `-=` `*=` `/=` `%=` `&=` `^=` `\|=` `<<=` `>>=` `>>>=` | 대입                     |
